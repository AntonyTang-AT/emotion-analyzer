# 项目代码结构规划（模块化开发指南）

本文档定义系统的代码组织方式，确保六层架构完全模块化、可插拔，并支持灵活的数据流控制。**本版本整合了“纯视觉旁路”、“两分支VA预测”和“L4动态路由权重”设计**，实现独立专家+动态路由思想。

## 二、模块化设计原则（更新）

### 2.1 每层独立与接口抽象

- 每层（L1~L6）均实现一个统一的 `run(input_data)` 接口，输入与输出为 `DataContext` 对象。
- 层内部可包含多个子模块，但对外暴露单一入口。
- 支持 **配置式启用/禁用**：通过 `pipeline.yaml` 可指定某些层跳过。

### 2.2 第一层模块化：每个特征提取器独立，并支持纯视觉旁路

- `feature_extractor` 基类定义 `extract(video_path, audio_path) -> FeatureDict`。
- 每个模态独立实现，注册到 factory。
- **新增约定**：对于视觉模态（宏表情、微表情），提取器需额外提供 `get_raw_visual(modal_name)` 方法或自动将原始特征存入 `DataContext.raw_visual_features`，供后续层直接使用。

### 2.3 第二层接口通用性（两分支）

- 预测器基类定义两个抽象方法：
  - `predict_self(features) -> VAConfidence`（自建模分支）
  - `predict_inter(features) -> VAConfidence`（交互分支）
- 通过 `registry` 将模态名映射到预测器实例。
- 预测器内部可共享底层网络，但顶层参数独立，分别训练或联合训练均可。

### 2.4 层间通信：统一数据上下文（DataContext）

上下文记录以下字段（新增字段已加粗）：

- `metadata`：会话ID、用户ID、时间戳、配置开关。
- `raw_data`：原始音频、视频路径。
- `features`：L1 输出（各模态特征）。
- **`raw_visual_features`**：纯视觉旁路特征（如 `micro_raw`, `macro_raw`）。
- **`va_self_predictions`**：L2 自建模分支输出。
- **`va_inter_predictions`**：L2 交互分支输出。
- `segments`：L3 输出的片段列表（每个片段包含两套VA）。
- `memory_retrieved`：L3 检索到的历史片段。
- **`contradiction`**：L4 输出（含 `suggested_fusion_weights`, `routing_confidence`）。
- `reports`：L5 输出。
- `personality`：L6 输出。

上下文可序列化为 JSON/MessagePack，便于缓存与调试。

## 三、关键模块详细说明（更新部分）

### 3.1 src/core/context.py

定义 `DataContext` 类，提供：
- `set_stage(stage_name, data)`
- `get_stage(stage_name)`
- `to_dict() / from_dict()`
- 路径管理（输入输出）

新增字段存储：
```python
self.raw_visual_features = {}   # {'micro': np.ndarray, 'macro': np.ndarray}
self.va_self_predictions = {}   # {'text': VAConfidence, ...}
self.va_inter_predictions = {}  # 同上
self.contradiction = None       # ContradictionResult 实例
```

### 3.2 src/layer2_predict/base_predictor.py（修改）

```python
class BasePredictor(ABC):
    @abstractmethod
    def predict_self(self, features: np.ndarray) -> VAConfidence:
        """自建模分支，不受其他模态影响，用于基线和冷启动"""
        pass

    @abstractmethod
    def predict_inter(self, features: np.ndarray) -> VAConfidence:
        """交互分支，特征与其他模态对齐，用于分段、融合和报告"""
        pass
```

`two_branch_mlp.py` 实现该基类。

### 3.3 src/layer2_predict/two_branch_mlp.py（新增）

- 共享底层：`Linear(feat_dim, 256) + ReLU`
- 自建模分支：`Linear(256, 128) + ReLU + Dropout + Linear(128, 3)` 输出 (v,a,conf)
- 交互分支：相同结构但独立参数
- 训练时可分别指定损失权重或使用多任务损失。

### 3.4 src/layer4_contradiction/contradiction_result.py（修改）

```python
@dataclass
class ContradictionResult:
    contradiction_type: str          # masking, sarcasm, hidden_emotion, intensity_mismatch, consistent
    contradiction_intensity: float   # max distance
    involved_modalities: List[str]   # 矛盾涉及模态对
    suggested_fusion_weights: List[float]  # 新增，4维，和为1
    routing_confidence: float        # 新增，0~1
```

### 3.5 src/layer4_contradiction/weight_selector.py（新增）

- 根据矛盾类型和可选的强度、置信度，生成 `suggested_fusion_weights`。
- 初版：查表（读取 `config/weight_table.yaml`）。
- 进阶：训练一个小型神经网络（输入：距离矩阵+各模态置信度+矛盾类型编码）。

`weight_table.yaml` 示例：
```yaml
masking: [0.1, 0.2, 0.2, 0.5]
sarcasm: [0.4, 0.4, 0.1, 0.1]
hidden_emotion: [0.1, 0.1, 0.1, 0.7]
intensity_mismatch: [0.3, 0.2, 0.3, 0.2]
consistent: [0.25, 0.25, 0.25, 0.25]
```

### 3.6 src/layer3_segment/memory_store.py（修改）

- `add_fragment()` 同时存储 `va_self_embedding` 和 `va_inter_embedding`（各8维）。
- `query_similar()` 接受参数 `embedding_type='inter'` 或 `'self'`，默认 `'inter'`。当 L4 输出矛盾且建议使用自建模时，可切换。

### 3.7 src/layer5_report/segment_reporter.py（修改）

- 接收 `context.contradiction.suggested_fusion_weights`。
- 构造 prompt 时，将权重转换为自然语言指令：“请重点关注微表情（权重0.5）和文本（权重0.3）的情绪变化”。

### 3.8 src/layer5_report/visualizer.py（新增函数）

- 原有 `plot_va_curve()` 使用加权平均后的 VA_inter 绘制曲线。
- 新增 `plot_weight_timeline(segments)` 绘制模态信任权重时序堆叠图。

### 3.9 src/layer6_personality/feature_aggregator.py（修改）

- 计算加权平均 VA：对每个片段，用 `suggested_fusion_weights` 加权 `va_inter_predictions`。
- 提取 `raw_visual_features` 的统计特征（均值、标准差、峰值、事件频率）。
- 提取 `routing_confidence` 的时序统计（均值、方差）。

## 四、层间通信示例（更新）

```
L1: 提取特征
    ├── features: text, speech, macro, micro
    └── raw_visual_features: micro_raw, macro_raw
        │
        ▼
L2: 两分支预测
    ├── va_self (各模态)
    └── va_inter (各模态)
        │
        ▼
L3: 分段 + 个性化
    ├── 综合唤醒度 ← va_inter
    ├── 基线校准 ← va_self
    └── 记忆检索 ← va_inter / va_self (按需)
        │
        ▼
L4: 矛盾检测 + 路由
    ├── 距离计算 ← va_inter
    ├── 规则分类
    └── 输出权重表 (contradiction.suggested_fusion_weights)
        │
        ▼
L5: 报告生成 ← 使用权重
L6: 人格回归 ← 使用权重 + 纯视觉特征
```

## 五、配置示例（pipeline.yaml，更新）

```yaml
pipeline:
  stages:
    L1:
      enabled: true
      feature_extractors: [text, speech, macro, micro]
      preserve_raw_visual: true   # 是否保存纯视觉旁路
    L2:
      enabled: true
      model_path: "./models/two_branch_va.pth"
      self_loss_weight: 0.3
      inter_loss_weight: 0.7
    L3:
      enabled: true
      segmentation:
        arousal_threshold: 0.3
        max_fragment_length: 30
        use_va_type: "inter"      # 分段使用交互分支
      baseline:
        enabled: true
        use_va_type: "self"       # 基线使用自建模分支
      cold_start:
        enabled: true
        embedding_source: "self"  # 冷启动使用自建模VA
      memory:
        db_path: "./data/user_db/chroma"
        decay_alpha: 0.1
    L4:
      enabled: true
      quadrant_thresholds: [0.50, 0.65, 0.70, 0.55]
      weight_strategy: "rule_table"   # rule_table / small_nn
      rule_table_path: "./config/weight_table.yaml"
    L5:
      enabled: true
      mode: "online"
      llm_model: "deepseek"
      use_l4_weights: true
    L6:
      enabled: true
      min_fragments: 30
      model_type: "lightgbm"
      use_weighted_stats: true
      use_raw_visual_features: true
```

## 六、扩展与维护建议（更新）

- **添加新模态**：在 `layer1_feature/` 新增提取器，在 `layer2_predict/` 注册两分支预测器。若为视觉模态，建议实现纯视觉旁路。
- **替换权重生成策略**：只需修改 `weight_selector.py`，不改变其他模块。
- **独立运行某一层**：提供脚本 `scripts/run_single_layer.py`，加载上下文、调用指定层并保存输出。
- **单元测试**：每个模块提供 `test_*.py`，验证接口正确性，特别是两分支输出和权重生成逻辑。
