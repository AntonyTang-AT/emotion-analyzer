# MER2026 经验迁移决策

> 来源：MER2026 Track2 实战总结。本文只记录 emotion-analyzer 的最终迁移决策；比赛仓库保持只读，不形成运行时依赖。

## 1. 迁移结论

六层架构、L1.4 微表情、L2 两分支 VA 与 L3 个性化保持不变。迁移重点是把 L4 从“矛盾类型直接查表”升级为“连续分歧驱动的选择性路由”，并严格分离 L4 预测与 L5 叙事。

## 2. 直接采用的工程经验

- 多专家输出必须经门控融合，禁止 naive union。
- 默认保持融合，仅对高分歧、高置信片段局部调整。
- 矛盾类型用于解释、统计和候选权重先验，不作为最终权重的唯一依据。
- 所有非默认融合必须写入 `fusion_audit_trail`。
- L5 只解释 L4 已锁定的加权 VA，禁止覆盖上游预测。
- 组件 val 指标不能替代独立 held-out 验证；阶段 7 前建立 D9-min。
- L6 必须使用 L4 最终加权 VA，不能简单等权聚合。

## 3. 只作冷启动参考的比赛参数

以下参数服务于 MER2026 的开放词汇 EW-F1，不是本项目的固定约束：

- SER `confidence_threshold=0.65`
- `max_switch_rate=0.10`
- DTRB/RRB 的标签数量上限
- openset Jaccard 分歧与 EW 同义词清洗

本项目将其映射为连续 VA 的置信门控、切换率统计与 `max_va_adjustment`，并使用 D9-min 重新标定。

## 4. AffectGPT 边界

AffectGPT checkpoint 仅作为可选 L2.A 并列专家：

- 输出 `reason_text` 与 openset 标签；
- reason 只作 L4 证据门和 L5 引用证据；
- 不替代 L1.4、L2 VA、L3 个性化或 L6 人格主路径；
- MVP 不接入，待 L1–L5 主路径和 D9-min 跑通后另立任务。

## 5. 本地报告模型决策

L5 默认使用本地 `Qwen2.5-7B-Instruct`（INT4，低温度短输出）。DeepSeek/OpenAI 保留为可选远程后端，不作为默认路径。模型失败或报告与加权 VA 极性冲突时降级为确定性模板摘要。

## 6. 实施顺序

1. 更新阶段 4、阶段 5、数据规划与本决策文档。
2. 新增 `config/fusion_policy.yaml`，配置改动独立 PR。
3. 扩展 L4 结果字段及序列化。
4. 实现分歧度量、规则解释、查表 fallback、选择性路由和局部修正。
5. 接入 L5 本地 Qwen，并执行预测/叙事边界检查。
6. 建立 D9-min，完成融合消融和参数标定。
7. MVP 后再评估 AffectGPT L2.A。

## 7. 禁止路线

- 外部 LLM 批量改写 reason、VA 或 openset。
- AffectGPT 端到端替换六层流水线。
- 无审计轨迹的静默融合修改。
- 仅凭验证集调参结果修改生产门控参数。
