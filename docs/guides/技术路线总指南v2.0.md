# 技术路线总指南（完整修改版）

## 整体架构总览

系统采用**六层流水线架构**：

| 层级 | 名称 | 核心任务 | 主要论文来源 |
|------|------|----------|----------------|
| **L1** | 多模态特征提取 | 原始视频/音频 → 各模态结构化特征 + **纯视觉旁路** | FRL-DGT, EmoLLM, Speech Attention |
| **L2** | 单模态VA预测 | 每模态独立预测两套VA：`VA_self`（自建模）和 `VA_inter`（交互对齐） | EmoLILMs, DTFE |
| **L3** | 自适应分段与个性化 | 动态切分、基线校准、冷启动、长效记忆 | CRM, Data-constrained, CDMER |
| **L4** | 矛盾检测 + 路由 | 模态间一致性度量 + 矛盾类型分类 + **输出融合权重** | CDMER, Real-Time MER |
| **L5** | 生成式报告 | 片段/整体自然语言报告 + 可视化（权重引导） | NUS-Emo, Dual-layer VR |
| **L6** | 人格回归 | 基于前五层输出推断大五人格分数（加权统计量 + 纯视觉特征） | 自研 + LightGBM |

整体数据流图（新增权重传递路径）：

```
原始视频/音频
  │
  ▼
L1 ──┬── 文本特征 ──→ L2_text ──┬── VA_self_text ──┐
     ├── 语音特征 ──→ L2_speech ──┼── VA_self_speech ──┤
     ├── 宏表情特征 ─→ L2_macro ──┼── VA_self_macro ──┤
     └── 微表情特征 ─→ L2_micro ──┴── VA_self_micro ──┐
          │                                           │
          └── 纯视觉旁路（原始特征）─────────────────→┤
                                                      ▼
                                                   L3/L4
                                                      │
                                                      ▼
                                              L4 输出：矛盾类型 + 融合权重
                                                      │
                                      ┌───────────────┼───────────────┐
                                      ▼               ▼               ▼
                                     L5报告         L3记忆加权      L6人格加权
```

## L1 层：多模态特征提取

### L1.1 文本特征提取

- **模型**：Whisper（base 或 small） + BERT-base-uncased
- **过程**：
  - Whisper 将音频流转换为带时间戳的句子文本。
  - BERT 对每个句子提取 [CLS] token 的 768 维嵌入。
- **输出**：(text_embedding: 768, start_time, end_time)

### L1.2 语音特征提取

- **模型**：Wav2Vec2-base + **遗忘注意力 (FoX)**
- **FoX 实现**：在 Transformer 编码层中引入可学习时域衰减系数 γ，对较早的 token 权重乘以 exp(-γ·Δt)，缓解长序列冗余。
- **辅助特征**：使用 librosa 提取基频 (F0)、能量、过零率、MFCC。
- **输出**：每秒一个语音特征向量（1024 维 Wav2Vec2 + 若干韵律特征）+ 时间戳。
- **论文来源**：Speech Emotion Recognition Attention Mechanisms

### L1.3 宏表情特征提取

- **模型**：MediaPipe Face Landmarker + VideoMAE-small
- **过程**：
  - MediaPipe 对人脸进行检测和对齐，得到 468 个关键点。
  - 裁剪人脸 ROI，缩放至 224×224。
  - VideoMAE 取连续 16 帧（步长 8 帧）输出 512 维时空特征，片段内平均池化。
- **输出**：每片段（或每秒）宏表情特征向量 (512 维) + 时间戳。

### L1.4 微表情特征提取（核心模块）

- **模型**：**DGM** + **多视角视觉投影** + **GCN**
- **DGM（位移生成模块）**：
  - 输入：检测到的微表情事件中的 **onset 帧** 和 **apex 帧**（灰度图 112×112）
  - 编码-解码器：5 层卷积 + 5 层反卷积，输出位移场 D ∈ R^{112×112×2}
  - 损失函数：L_total = L_recon + λ_smooth·L_smooth + λ_norm·L_norm
    - L_recon = ||warp(I_onset, D) - I_apex||_1（扭曲重建损失）
    - L_smooth：TV 正则，L_norm：位移幅度正则。
- **多视角视觉投影**：
  - 内容视角：对 DGM 中间层特征进行 K-Means 聚类，取聚类中心。
  - 关系视角：以人脸关键点为节点，节点特征为局部区域特征，用 2 层 GCN 提取结构关系。
  - 融合：内容特征与关系特征拼接后经全连接层 → 256 维。
- **弱信号放大**：对关键 AU（AU12、AU15 等）的原始激活值乘以系数 2~3（来自 FRL-DGT）。
- **输出**：每个微表情事件一个 256 维特征向量 + 时间戳。
- **论文来源**：FRL-DGT, EmoLLM

- **新增：纯视觉旁路（借鉴 Libra）**  
  在微表情特征提取过程中，将 GCN 输出前的原始 256 维特征（或中间层特征）**额外保存一份**到数据上下文的 `raw_visual_features['micro']` 字段。该旁路**不经过 L2 预测器**，直接供 L4 矛盾检测、L5 报告、L6 人格回归使用，以避免微表情细节被文本/语音模态稀释，特别用于 `hidden_emotion` 等矛盾场景。

## L2 层：单模态VA预测（两分支架构）

### L2.1 独立预测模型（改进为两分支）

每个模态一个 **两分支轻量 MLP**：

- 输入维度：文本 768，语音 1024，宏表情 512，微表情 256。
- 共享底层：Linear + ReLU → 256 维公共表示。
- 分叉：
  - **自建模分支（self）**：2 层 MLP（128→128）+ ReLU + Dropout(0.2)，输出 (valence_self, arousal_self, confidence_self)
  - **交互分支（inter）**：2 层 MLP（128→128）+ ReLU + Dropout(0.2)，输出 (valence_inter, arousal_inter, confidence_inter)
- 输出范围：均在 [-1,1] 或 [0,1] 统一。

**训练方式**：
- 自建模分支可使用单模态 VA 标注数据（如仅微表情预测 VA）或自监督预训练，损失函数 MSE + 二元交叉熵。
- 交互分支使用多模态对齐数据（完整视频 + VA 标注）训练。
- 初期可用同一数据集同时训练两个分支，但需在损失函数中区分（可设置不同权重）。

### L2.2 多任务联合优化（EmoLILMs）适配

- 共享表示空间仅应用于 **交互分支** 的投影，自建模分支保持独立。
- 可学习损失权重：L_total = λ_v·L_v + λ_a·L_a + λ_c·L_c，其中 λ 为可学习标量。
- 输出：每个模态的 (v_self, a_self, conf_self) 和 (v_inter, a_inter, conf_inter)，以及共享表示（供 L3/L4 使用）。

### L2.3 可选增强：DTFE（后期优化）

- 使用动态Token特征提取器对长序列语音/微表情进行 token 化，通过跨序列交互(CSI)和时序频谱建模(ISI)提升表示质量。初期可跳过。
- 论文来源：Structured multi-modal representation learning (DTFE)

## L3 层：自适应分段与个性化

### L3.1 动态分段控制器

- **输入**：每秒四模态 **VA_inter** 值（来自 L2 交互分支，已对齐到 1 秒粒度）
- **算法**：
  - 计算每秒综合唤醒度 A_combined = Σ(conf_inter_i * A_inter_i) / Σ(conf_inter_i)
  - 唤醒度变化率 ΔA(t) = |A_combined(t) - A_combined(t-1)|
  - 愉悦度极性翻转：sign(V_inter(t)) ≠ sign(V_inter(t-1))
  - **切分条件**（满足任一）：
    - ΔA(t) > 0.3
    - 极性翻转
    - 模态间最大距离 > 0.6（前期可注释）
  - 若片段长度超过 30 秒仍未触发，强制切分。
- **可选优化（CRM窗口选择）**：在每个时间点，对候选窗口长度 (2-30秒) 计算四模态 VA_inter 平均距离，选择使距离最小的窗口长度作为最优分割点。
- **输出**：片段列表 [{id, start, end, frames_VA_inter}]
- **论文来源**：Continuous Estimation of Emotions in Speech (CRM)

### L3.2 静态基线校准（Data-constrained）

- **触发条件**：用户首次使用且同意校准。
- **过程**：
  - 播放 3-5 段标准情感刺激视频（高兴、悲伤、恐惧各 1 段，每段 15 秒）。
  - 采集用户观看时的四模态 **VA_self** 响应（片段平均值），因为自建模分支更能反映用户真实表达风格。
  - 已有标准人群基线（可从公开数据集或预先采集的 30+ 用户获得）。
  - 计算偏移量：ΔVA = (ΔV, ΔA)，每个模态独立。
- **存储**：用户ID + ΔVA 存入本地 SQLite/JSON。
- **应用**：后续所有单模态 **VA_self** 预测减去 ΔVA。

### L3.3 合作式冷启动（CRM）

- **触发**：新用户无基线数据时。
- **过程**：
  - 已有用户库存储每个用户的"首次会话平均 **VA_self** 向量"（8维：4模态 × 2）。
  - 新用户完成第一个片段后，计算其平均 VA_self 向量。
  - 余弦相似度，取 Top-5 相似用户。
  - 加权融合：ΔVA_new = Σ(sim_i * ΔVA_i) / Σ(sim_i)
- **输出**：临时 ΔVA，随用户交互增加逐步被真实 ΔVA 替代。

### L3.4 长效记忆模块

- **向量数据库**：Chroma 或 FAISS
- **存储内容**：每个片段一条记录，字段：
  - user_id
  - timestamp
  - embedding_self：8 维（四模态 VA_self 值）
  - embedding_inter：8 维（四模态 VA_inter 值）
  - metadata：{emotion_label, contradiction_type, au_pattern}
- **检索**：
  - 当前片段特征向量 q（默认为 8 维 VA_inter，但当 L4 权重指示矛盾时可选择使用 VA_self）。
  - 计算与历史片段的余弦相似度，再乘以时间衰减权重 ω = exp(-α·(current_time - timestamp))，α 取 0.1。
  - 返回 Top-3 最相似片段及其 metadata。
- **论文来源**：Data-constrained personalization (时间衰减), Real-Time MER (连续学习)

### L3.5 域适应对齐（RSTR，可选）

- **目的**：减小不同用户之间的特征分布差异。
- **方法**：在 L3 特征后加一层可学习线性变换（域适应层），用少量新用户数据微调该层参数。
- **实现**：借鉴 CDMER 的区域选择性迁移回归思想，仅对关键特征维度进行适配。
- **论文来源**：Cross-Database Micro-Expression Recognition (CDMER)

## L4 层：矛盾检测 + 动态路由

### L4.1 VA空间距离计算

- **输入**：当前片段中四模态的 (v_inter, a_inter, conf_inter)
- **计算**：对每对模态 (i,j) 欧氏距离 d_ij = sqrt((v_i - v_j)^2 + (a_i - a_j)^2)
- **输出**：4×4 距离矩阵（对角线为 0）。

### L4.2 象限分组阈值（QBTD）

- 根据片段综合 VA（加权平均，使用 VA_inter）所在象限确定阈值。
- 若任何 d_ij > thr_quadrant，则认为存在矛盾。
- **论文来源**：Continuous Estimation of Emotions in Speech (QBTD)

### L4.3 专家规则分类（保持不变）

规则优先级：masking > sarcasm > hidden_emotion > intensity_mismatch

规则实现（Python）：
```python
def classify_contradiction(text_v, speech_v, macro_v, micro_v):
    if text_v > 0.6 and micro_v < -0.4: return "masking"
    if text_v > 0.3 and (speech_v < -0.4 or macro_v < -0.4): return "sarcasm"
    if abs(macro_v) < 0.2 and abs(micro_v) > 0.5: return "hidden_emotion"
    if same_sign(text_v, micro_v) and abs(text_v - micro_v) > 0.6: return "intensity_mismatch"
    return "consistent"
```

- **矛盾强度**：所有距离中的最大值 max(d_ij)。
- **涉及模态对**：产生最大距离的模态对。

### L4.4 新增：融合权重生成（动态路由）

为每个矛盾类型预先定义一组融合权重 `suggested_fusion_weights`（4维，和为1），以及该权重的置信度 `routing_confidence`。

**初版策略（规则查表）**：

| 矛盾类型 | 文本权重 | 语音权重 | 宏表情权重 | 微表情权重 | 说明 |
|----------|----------|----------|------------|------------|------|
| masking | 0.1 | 0.2 | 0.2 | **0.5** | 微表情泄露真实情绪 |
| sarcasm | 0.4 | 0.4 | 0.1 | 0.1 | 文本和语音最重要 |
| hidden_emotion | 0.1 | 0.1 | 0.1 | **0.7** | 微表情核心 |
| intensity_mismatch | 0.3 | 0.2 | 0.3 | 0.2 | 平均信任 |
| consistent | 0.25 | 0.25 | 0.25 | 0.25 | 等权 |

- **routing_confidence** 可设为矛盾强度的归一化值（如 min(1.0, intensity/1.2)），或基于模态置信度均值。
- **进阶方案**：训练一个小型神经网络（输入：距离矩阵 + 各模态置信度 + 矛盾类型 one-hot，输出权重），初期可跳过。

**输出结构**（ContradictionResult）：
```json
{
  "contradiction_type": "masking",
  "contradiction_intensity": 0.72,
  "involved_modalities": ["text", "micro"],
  "suggested_fusion_weights": [0.1, 0.2, 0.2, 0.5],
  "routing_confidence": 0.85
}
```

- **论文来源**：Real-Time Micro-Expression Recognition with AU+GPT（以及借鉴 Libra 多模态大模型路由思想）

## L5 层：生成式报告

### L5.1 片段级报告

- **模型**：DeepSeek-V3 或 GPT-4o-mini（也可用本地 Qwen-7B-Chat）
- **Prompt 模板**（参考 NUS-Emo，新增权重引导）：
  ```
  你是一个情感分析专家。根据以下信息生成30-80字的片段总结：
  - 各模态重要性权重：文本{w_text}，语音{w_speech}，宏表情{w_macro}，微表情{w_micro}
  - 综合VA轨迹（加权后）：从({v1},{a1})到({v2},{a2})
  - 矛盾类型：{contradiction_type}
  - 历史相似片段（Top-2）：{similar_history}
  请重点关注权重最高的模态所反映的情绪变化。
  ```
- **输出**：自然语言文本，存入片段结果。

### L5.2 整体级报告

- **输入**：所有片段报告 + 整段 VA_inter 时序 + 矛盾事件汇总 + 各片段权重。
- **Prompt 模板**：
  ```
  请基于以下片段的情绪分析，给出整个对话的总结（150-300字）：
  [片段1报告]
  [片段2报告]
  ...
  并计算平静指数（1-10）和焦虑指数（1-10）。
  ```
- **高阶映射（Dual-layer VR）**：
  - 焦虑指数 = (恐惧片段占比×0.4 + 悲伤占比×0.3 + 愤怒占比×0.2 + 惊讶占比×0.1) × 10
  - 平静指数 = 10 - 焦虑指数（近似）
- **论文来源**：NUS-Emo, Dual-Layer Emotion Sensing in VR

### L5.3 可视化输出

- **VA时序曲线**：使用 plotly 或 matplotlib，横轴时间，纵轴 valence（红）/ arousal（蓝）。曲线基于加权平均后的 VA_inter（使用各片段权重）。
- **模态一致性热力图**：seaborn.heatmap 绘制 4×4 矩阵，对角线为 1.00，非对角为当前片段的距离（归一化到 [0,1]）。
- **新增：模态信任权重时序图**：堆叠面积图，展示每个片段各模态的 `suggested_fusion_weights`，便于观察矛盾场景下系统对不同模态的信任变化。
- **统计摘要模板**：
  - "高唤醒状态占比约{X}%"
  - "检测到矛盾{N}处，其中伪装{mask}处，讽刺{sarc}处"

## L6 层：人格回归（大五人格）

### L6.1 特征抽取（从 L1-L4 结果中聚合，改进版）

- **时序统计**：整个对话中 **加权平均后的 VA**（使用每个片段的 `suggested_fusion_weights` 对 VA_inter 加权）的均值、标准差、偏度、自相关系数（滞后1-5）。
- **矛盾统计**：各类矛盾次数、占比、强度均值，以及平均路由置信度、各矛盾类型下的平均权重分布。
- **语音行为**：平均语速（字/秒）、音调标准差、微笑 AU12 强度、微表情恐惧次数。
- **响应延迟**：平均回答延迟时间（从问题结束到开始回答）。
- **个性化偏移**：最终采用的 ΔVA 向量（基于 VA_self）。
- **纯视觉特征**：直接使用 L1 的纯视觉旁路特征（如微表情原始特征的平均强度、事件频率、DGM 位移场能量等）。
- **特征数量**：约 30-60 维（依实际提取）。

### L6.2 回归模型

- **主模型**：多输出 LightGBM
  - 输入：标准化后的特征向量
  - 输出：5 个连续值（范围 1-10），对应 O, C, E, A, N
  - 损失：MAE + 可选的多输出相关性正则项（λ * Σ|ρ_pred - ρ_true|）
- **概率增强**（可选）：Gaussian Process (GP) 或贝叶斯神经网络，输出均值 + 方差。
- **冷启备选**：当特征不足时，调用大语言模型（GPT-4o）进行零样本人格推断，Prompt 包含行为特征摘要。

### L6.3 输出格式

```json
{
  "personality": {
    "O": 7.2, "C": 6.8, "E": 5.1, "A": 8.0, "N": 3.2
  },
  "confidence_interval": [0.7, 0.6, 0.8, 0.7, 0.9],
  "behavioral_evidence": "快速语速、频繁微笑，矛盾类型以隐含情绪为主，微表情强度较高"
}
```

### L6.4 训练数据

- **首选**：自建小样本（30-50 人，完成 BFI-10 量表 + 标准化对话），获取特征-人格标签对。
- **弱标注**：使用 Friends 角色的人格研究文献作为先验，抽取角色多轮对话生成伪标签。
- **半自动**：先用 GPT-4o 对无标签对话进行零样本人格标注，再在人工标注子集上微调。

### L6.5 评估指标

- MAE（目标 ≤ 0.8）
- Pearson 相关系数（目标 ≥ 0.6）
- ICC（重测信度，目标 ≥ 0.7）

## 附录：各层使用的具体模型与库汇总（更新）

| 层级 | 模型/算法 | 主要库 | 备注 |
|------|-----------|--------|------|
| L1 文本 | Whisper + BERT | openai-whisper, transformers | 离线运行 |
| L1 语音 | Wav2Vec2 + FoX | transformers, torch | FoX 需自定义层 |
| L1 宏表情 | MediaPipe + VideoMAE | mediapipe, transformers | VideoMAE 可用 ResNet+GRU 替换 |
| L1 微表情 | DGM + GCN + **纯视觉旁路** | torch, dgl | DGM 需自实现，初期可用 TV-L1 光流；旁路保存原始特征 |
| L2 单模态 | **两分支 MLP** (self+inter) + 多任务损失 | torch | 共享底层，顶层独立 |
| L3 分段 | 规则 + CRM窗口 | numpy | CRM 可选 |
| L3 记忆 | Chroma/FAISS | chromadb, faiss | 存储两套 embedding |
| L4 矛盾 | 距离计算 + 规则 + **权重查表** | numpy | 输出增加权重和置信度 |
| L5 报告 | DeepSeek/GPT + **权重引导** | openai, requests | 也可用本地 Qwen |
| L6 人格 | LightGBM / GP + **加权统计量** | lightgbm, scikit-learn | GP 用 GPy 或 sklearn |
