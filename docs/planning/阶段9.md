  搭建基础HTML页面（登录页、主页、上传页、实时摄像头页），实现简单的单页应用（SPA）效果或分页面跳转，配置静态资源目录。

- **基本内容**  
  创建 `web/index.html`（主页/上传页）、`web/login.html`（登录页）、`web/camera.html`（实时摄像头页），以及公共的CSS和JS文件。实现基础页面结构和导航。

- **任务实施的思路和结构**  
  1. 使用原生HTML/CSS/JS，无框架（或可选轻量框架如Alpine.js），便于部署。

### 任务9.4：实现VA曲线图（加权后）

- **任务题目**  
  使用 Chart.js 或 ECharts 绘制基于加权后的VA时序曲线（效价、唤醒度随时间变化）。纵轴Valence/Arousal（范围-1到1），横轴时间（秒或片段索引）。曲线应使用L4输出的 `suggested_fusion_weights` 对 `va_inter` 加权平均。

- **基本内容**  
  在 `web/js/charts.js` 中实现 `drawVACurve(segments)`。每个片段包含时间范围和各模态 `va_inter` 序列。需先计算每个时间点的加权综合VA（使用该片段对应的融合权重），然后绘制两条曲线（valence红色，arousal蓝色）。

- **任务实施的思路和结构**  
  1. 输入：`segments` 数组，每个元素含 `start_time`, `end_time`, `va_inter`（各模态按时间点的数组），以及 `contradiction.suggested_fusion_weights`。  
  2. 对每个片段内的时间点，计算加权综合VA：  
     `v_weighted = Σ(weight_i * v_i)`，`a_weighted` 同理，其中 `i` 遍历模态。  
  3. 构建 `[time, valence, arousal]` 列表。  
  4. 使用 ECharts 或 Chart.js 绘制双线图，添加图例、网格、工具提示。  
  5. 可选择显示片段边界（虚线）。  
  6. 返回图表实例，支持缩放。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`web/js/charts.js`  
  - 参考：`接口与web端相关项目结构V2.0.docx` 中 `charts.js` 和 `drawWeightChart`；`技术路线总指南V2.0.docx` 中 VA曲线绘制要求。

- **需要参考的具体论文或者模型**  
  无。

- **预估时间（AI/agent辅助）**

### 任务9.5：实现模态一致性热力图

- **任务题目**  
  绘制模态间VA距离的4×4热力图，展示各模态在整体会话中的平均不一致性（或每个片段可切换）。初版可对整个会话平均距离矩阵绘制一个热力图。

- **基本内容**  
  在 `charts.js` 中添加 `drawConsistencyHeatmap(segments)`。计算所有片段距离矩阵的平均值（4×4），使用 `seaborn` 风格或 ECharts 热力图展示，颜色越红表示距离越大（不一致性强）。

- **任务实施的思路和结构**  
  1. 遍历所有片段，对每个片段调用后端距离计算（或前端自己根据 `va_inter` 计算每对模态欧氏距离）。  
  2. 累加距离矩阵，最后除以片段数得到平均距离。  
  3. 对角线设为1（自距离归一化）或保留0。  
  4. 使用 ECharts 的热力图或 Chart.js 的矩阵渲染（可借助 `heatmap.js` 或手动 canvas）。简单用 ECharts。  
  5. 展示模态标签（文本、语音、宏表情、微表情）。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 修改：`web/js/charts.js`  
  - 参考：`技术路线总指南V2.0.docx` 中 L5.3 热力图描述。

### 任务9.8：实现实时摄像头模式（WebSocket 集成）

- **任务题目**  
  实现实时摄像头页面，获取用户摄像头视频流，每秒抽取一帧发送到 WebSocket 端点，实时绘制 VA 曲线，结束时发送结束信号并接收最终报告与权重图。

- **基本内容**  
  在 `web/js/camera.js` 中实现：  
  - 获取 `getUserMedia` 视频流，展示在 `<video>` 元素。  
  - 使用 Canvas 定时捕获帧（如每秒1帧），转换为 base64 或 blob，通过 WebSocket 发送。  
  - 接收服务器返回的 `{valence, arousal, confidence}` 实时点，绘制实时 VA 曲线（不断追加）。  
  - 用户点击“停止”时，发送 `{end: true}`，等待服务器返回最终结果（报告、权重时序等），并调用绘图函数展示。

- **任务实施的思路和结构**  
  1. 建立 WebSocket 连接：`ws://localhost:8000/api/v1/ws/analyze?token=xxx`。  
  2. 监听 `onmessage`，解析 JSON：若为实时点，更新曲线；若为最终结果，渲染报告和权重图。  
  3. 曲线使用动态数据更新（可用 Chart.js 动态添加数据）。  
  4. 帧捕获频率可配置（如1fps），注意性能。  
  5. 显示连接状态（已连接、分析中、完成）。  
  6. 错误处理和重连。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`web/js/camera.js`, 修改 `web/camera.html`  
  - 参考：`接口与web端相关项目结构V2.0.docx` 中 `camera.js` 描述。
