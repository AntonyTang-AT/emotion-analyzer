### 任务1.1：文本特征提取器

- **任务题目**  
  实现文本特征提取器（Whisper + BERT）

- **基本内容**  
  编写 `src/layer1_feature/text_extractor.py`，实现从原始音频中提取带时间戳的句子文本，并使用 BERT 提取每个句子的 768 维嵌入向量。

- **任务实施的思路和结构**  
  1. 继承 `FeatureExtractor` 基类（已在阶段0定义）。  
  2. 使用 `whisper` 库（openai-whisper）加载 base 或 small 模型，将音频转换为带时间戳的句子列表。  
  3. 对每个句子，使用 `transformers` 库的 `BertTokenizer` 和 `BertModel`（base-uncased），提取 `[CLS]` token 的最后一层隐藏状态（768维）。  
  4. 输出格式：列表，每个元素为 `(text_embedding: np.ndarray (768,), start_time: float, end_time: float)`。  
  5. 集成到 `DataContext.features['text']` 中。  
  6. 配置化：Whisper 模型大小、BERT 模型名称可从 `config/models.yaml` 读取。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer1_feature/text_extractor.py`  
  - 修改：`src/layer1_feature/__init__.py`（导出）  
  - 参考：`技术路线总指南V2.0.docx` 中 L1.1 章节；`具体核心项目结构V2.0.docx` 中 `text_extractor.py` 的提及

- **需要参考的具体论文或者模型**  
  - Whisper: `Robust Speech Recognition via Large-Scale Weak Supervision` (OpenAI, 2022)  
  - BERT: `BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding` (Devlin et al., 2018)  
  - 无直接论文总览中的条目，属于标准工具。

- **预估时间（AI/agent辅助）**  
  1.5 小时（模型下载时间除外）

### 任务1.2：语音特征提取器（含遗忘注意力FoX）

- **任务题目**  
  实现语音特征提取器（Wav2Vec2 + 遗忘注意力FoX + 韵律特征）

- **基本内容**  
  编写 `src/layer1_feature/speech_extractor.py`，使用 Wav2Vec2 提取帧级特征，并在 Transformer 层中引入遗忘注意力（FoX）机制以缓解长序列冗余；同时使用 librosa 提取基频（F0）、能量、过零率、MFCC 等韵律特征，最终输出每秒一个特征向量。

- **任务实施的思路和结构**  
  1. 加载预训练 Wav2Vec2 模型（如 `facebook/wav2vec2-base`）。  
  2. **FoX 实现**：继承或修改 Wav2Vec2 的 Transformer 编码层，在自注意力计算后引入时域衰减系数 γ，对较早的 token 权重乘以 `exp(-γ·Δt)`。可自定义一个 `FoXWav2Vec2Model` 类，或在特征提取后独立计算衰减（更简单）。  
     - 简化方案：提取 Wav2Vec2 输出的帧级特征序列后，再应用指数衰减权重。  
  3. 使用 `librosa` 提取每秒的韵律特征：F0（pyin）、能量（rms）、过零率、13 维 MFCC。  
  4. 将 Wav2Vec2 输出（1024维）与韵律特征拼接，形成每秒特征向量（维度 ≈ 1024 + 1+1+1+13 = 1040）。  
  5. 输出格式：列表，每个元素为 `(speech_feature: np.ndarray, timestamp: float)`，timestamp 为秒级。  
  6. 集成到 `DataContext.features['speech']`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer1_feature/speech_extractor.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L1.2 章节；`论文总览.docx` 中 `Speech Emotion Recognition Attention Mechanisms` 论文的遗忘注意力（FoX）描述

- **需要参考的具体论文或者模型**  
  - **Wav2Vec2**: `wav2vec 2.0: A Framework for Self-Supervised Learning of Speech Representations` (Baevski et al., 2020)  
  - **遗忘注意力（FoX）**: 来自 `Speech Emotion Recognition Attention Mechanisms` 论文中提到的 FoX（Forgetting Attention）机制，可参考原文实现。  
  - 韵律特征提取：可参考 librosa 文档，无特定论文。

- **预估时间（AI/agent辅助）**  
  2 小时（包含 FoX 的自定义实现）

### 任务1.3：宏表情特征提取器

- **任务题目**  
  实现宏表情特征提取器（MediaPipe + VideoMAE）

- **基本内容**  
  编写 `src/layer1_feature/macro_extractor.py`，使用 MediaPipe 进行人脸检测与对齐，裁剪 ROI 后输入 VideoMAE 提取时空特征，输出每片段（或每秒）的 512 维特征向量。

- **任务实施的思路和结构**  
  1. 使用 `mediapipe` 库加载 `Face Landmarker` 模型，对视频每一帧检测 468 个关键点，得到人脸包围框，进行对齐和裁剪，缩放到 224×224。  
  2. 使用 `transformers` 库的 `VideoMAE` 模型（如 `MCG-NJU/videomae-base-finetuned-kinetics`）或自己实现简化版（用 ResNet+GRU 作为备选）。  
  3. 取连续 16 帧（步长 8 帧）作为一个片段，输入 VideoMAE，输出 512 维时空特征，对片段内平均池化。  
  4. 将特征对齐到每秒（若片段与秒不严格对齐，可采用线性插值或最近邻分配）。  
  5. 输出格式：列表，每个元素为 `(macro_feature: np.ndarray (512,), timestamp: float)`。  
  6. 集成到 `DataContext.features['macro']`。  
  7. 配置化：是否使用 VideoMAE 或简化模型、帧采样参数。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer1_feature/macro_extractor.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L1.3 章节；`具体核心项目结构V2.0.docx` 中 `macro_extractor.py`

- **需要参考的具体论文或者模型**  
  - **MediaPipe Face Landmarker**: Google MediaPipe 技术报告  
  - **VideoMAE**: `VideoMAE: Masked Autoencoders are Data-Efficient Learners for Self-Supervised Video Pre-Training` (Tong et al., 2022)  
  - 备选方案（ResNet+GRU）：无特定论文，属于经典架构。

- **预估时间（AI/agent辅助）**  
  2 小时（包含模型加载和视频帧处理）

### 任务1.4：微表情特征提取器（核心模块）

- **任务题目**  
  实现微表情特征提取器（DGM + 多视角视觉投影 + GCN）

- **基本内容**  
  编写 `src/layer1_feature/micro_extractor.py`，实现位移生成模块（DGM）学习 onset 到 apex 帧的位移场；通过多视角视觉投影（内容视角 K-Means + 关系视角 GCN）融合特征；输出每个微表情事件的 256 维特征向量。同时实现弱信号放大（关键 AU 激活值乘以 2~3 倍）。

- **任务实施的思路和结构**  
  1. **微表情事件检测**：可使用现有方法（如基于光流或 AU 强度变化）检测 onset 和 apex 帧，初期可简化：手动标注或使用 OpenFace 提取 AU 强度后阈值判断。  
  2. **DGM 实现**：  
     - 输入：onset 帧和 apex 帧（灰度图 112×112）。  
     - 编码器：5 层卷积（通道：64→128→256→512→512），输出位移场 D（112×112×2）。  
     - 解码器：5 层反卷积，重建 apex 帧。  
     - 损失函数：`L_recon = L1_loss(warp(I_onset, D), I_apex)` + `L_smooth`（TV 正则） + `L_norm`（位移幅度正则）。  
     - 可预先训练 DGM 模型，或使用 TV-L1 光流作为替代（初期简化）。  
  3. **多视角视觉投影**：  
     - 内容视角：对 DGM 中间层特征图进行 K-Means 聚类（k=16），取聚类中心作为内容特征。  
     - 关系视角：以 MediaPipe 468 个关键点为节点，每个节点的特征为局部区域（如 5×5）的特征，使用 2 层 GCN（图卷积网络）提取 128 维关系特征。  
     - 融合：拼接内容特征（128维）和关系特征（128维）→ 256 维。  
  4. **弱信号放大**：对关键 AU（如 AU12、AU15）的原始激活值乘以系数 2~3（可从 OpenFace 或 MediaPipe 的 AU 强度获得）。  
  5. 输出格式：每个微表情事件一个 `(micro_feature: np.ndarray (256,), start_time, end_time)`。  
  6. 集成到 `DataContext.features['micro']`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer1_feature/micro_extractor.py`，以及可能需要的子模块（`dgm.py`, `gcn.py`）  
  - 参考：`技术路线总指南V2.0.docx` 中 L1.4 章节；`论文总览.docx` 中 **FRL-DGT** 论文的核心方法

- **需要参考的具体论文或者模型**  
  - **FRL-DGT**: `Facial Micro-Expression Recognition with Displacement Generation and Transformer-based Fusion`（提供 DGM、多级融合、弱信号放大）  
  - **GCN**: `Semi-Supervised Classification with Graph Convolutional Networks` (Kipf & Welling, 2016)  
  - **多视角视觉投影**: 参考 `EmoLLM` 论文中的内容视角+关系视角分解

- **预估时间（AI/agent辅助）**  
  3 小时（包含 DGM 模型搭建和 GCN 实现，若使用 TV-L1 光流可降至 1.5 小时）

### 任务1.5：纯视觉旁路（保存原始特征）

- **任务题目**  
  实现纯视觉旁路：在微表情特征提取过程中保存原始特征

- **基本内容**  
  在微表情提取器中，额外将 GCN 输出前的原始 256 维特征（或中间层特征）保存到 `DataContext.raw_visual_features['micro']`，不经过 L2 预测器，供 L4 矛盾检测、L5 报告、L6 人格回归直接使用，以避免微表情细节被其他模态稀释。

- **任务实施的思路和结构**  
  1. 在 `micro_extractor.py` 的 `extract()` 方法中，计算得到最终的 256 维微表情特征后，同时将其副本存入 `context.raw_visual_features['micro']`。  
  2. 可选：也保存宏表情的原始特征（`raw_visual_features['macro']`）供对比。  
  3. 确保 `DataContext` 已定义 `raw_visual_features` 字段（阶段0已做）。  
  4. 在配置 `pipeline.yaml` 中增加开关 `preserve_raw_visual: true`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 修改：`src/layer1_feature/micro_extractor.py`，`src/layer1_feature/macro_extractor.py`（可选）  
  - 参考：`技术路线总指南V2.0.docx` 中的“新增：纯视觉旁路”章节；`具体核心项目结构V2.0.docx` 中 `raw_visual_features` 字段

- **需要参考的具体论文或者模型**  
  - **Libra**（文档中提到）：多模态大模型路由思想，强调保留原始视觉特征。论文总览中未包含，可参考 `Libra: Leveraging Visual Signals for Multimodal Sentiment Analysis` 等。

- **预估时间（AI/agent辅助）**  
  0.2 小时（简单复制赋值）

### 任务1.6：特征提取工厂（factory.py）

- **任务题目**  
  实现特征提取工厂，支持按配置开关各模态提取器

- **基本内容**  
  编写 `src/layer1_feature/factory.py`，根据 `pipeline.yaml` 中的 `feature_extractors` 列表动态实例化所需的特征提取器，并提供一个统一的 `extract_all(context)` 函数，依次调用各提取器，将结果存入 `DataContext.features` 和 `raw_visual_features`。

- **任务实施的思路和结构**  
  1. 定义 `FeatureExtractorFactory` 类，包含 `register(name, extractor_class)` 和 `create(name)` 方法。  
  2. 在 `__init__.py` 中注册所有提取器（text, speech, macro, micro）。  
  3. 实现 `get_extractors_from_config(config)` 读取 `pipeline.yaml` 中 L1 的 `feature_extractors` 列表，返回实例列表。  
  4. 实现 `run_l1(context)` 函数，遍历提取器，调用 `extract(context)`，将特征存入 `context.features[modal_name]`，同时若提取器提供 `get_raw_visual()` 则存入 `context.raw_visual_features`。  
  5. 异常处理：某个提取器失败时记录日志并跳过。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer1_feature/factory.py`  
  - 修改：`src/layer1_feature/__init__.py`（注册）  
  - 参考：`具体核心项目结构V2.0.docx` 中 `factory.py` 的提及
