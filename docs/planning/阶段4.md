### 任务4.1：实现VA空间距离计算模块

- **任务题目**  
  实现模态间VA空间距离计算，用于衡量文本、语音、宏表情、微表情两两之间的不一致程度。

- **基本内容**  
  编写 `src/layer4_contradiction/va_distance.py`，输入一个片段内四个模态的 `(valence, arousal)` 预测值（使用 `va_inter`），计算每对模态的欧氏距离，输出4×4距离矩阵。

- **任务实施的思路和结构**  
  1. 输入格式：字典 `{modal: (v, a)}`，模态名为 `text`, `speech`, `macro`, `micro`。  
  2. 计算每对 `(i, j)` 的欧氏距离：`d_ij = sqrt((v_i - v_j)^2 + (a_i - a_j)^2)`。  
  3. 输出为4×4的 `numpy.ndarray`，对角线为0。  
  4. 同时可返回最大距离 `max_distance` 以及最大距离对应的模态对 `(modal_i, modal_j)`。  
  5. 该模块被L4主流程调用。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer4_contradiction/va_distance.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L4.1 章节。

- **需要参考的具体论文或者模型**  
  - 无特定论文，基础距离计算。

- **预估时间（AI/agent辅助）**  
  0.3 小时

### 任务4.2：实现象限分组阈值矛盾判定（QBTD）

- **任务题目**  
  实现基于VA象限的动态阈值矛盾判定，根据当前片段的综合VA所在象限，选择不同的距离阈值来判断是否存在矛盾。

- **基本内容**  
  编写 `src/layer4_contradiction/quadrant_threshold.py`，根据片段综合VA（加权平均后的 valence 和 arousal）所在的象限（Q1~Q4），从配置中读取对应阈值 `thr_quadrant`，若模态间最大距离超过该阈值，则判定存在矛盾。输出布尔值及矛盾强度。

- **任务实施的思路和结构**  
  1. 计算片段综合VA：使用 `va_inter` 的各模态平均（或加权，权重初期等权）。  
  2. 根据 valence 和 arousal 的符号划分象限：  
     - Q1: v>0, a>0 (高愉悦高唤醒)  
     - Q2: v<0, a>0 (低愉悦高唤醒)  
     - Q3: v<0, a<0 (低愉悦低唤醒)  
     - Q4: v>0, a<0 (高愉悦低唤醒)  
  3. 从配置 `config/pipeline.yaml` 的 `quadrant_thresholds` 数组中读取对应阈值（顺序 Q1~Q4）。  
  4. 比较 `max_distance`（来自任务4.1）与阈值，若超过则 `has_contradiction = True`，矛盾强度 `intensity = max_distance`。  
  5. 该模块可选，也可直接使用后续专家规则分类（专家规则内部也会判断是否存在矛盾）。初期可简化：不单独使用QBTD，仅作为矛盾强度归一化的参考。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer4_contradiction/quadrant_threshold.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L4.2 章节；`论文总览.docx` 中 **Continuous Estimation of Emotions in Speech**（象限分段优化QBTD）。

- **需要参考的具体论文或者模型**  
  - **Continuous Estimation of Emotions in Speech**：象限分段优化（QBTD），按VA象限分别设置阈值。

- **预估时间（AI/agent辅助）**  
  0.5 小时

### 任务4.3：实现专家规则分类器（矛盾类型判定）

- **任务题目**  
  实现基于阈值的专家规则分类器，输出矛盾类型：`masking`, `sarcasm`, `hidden_emotion`, `intensity_mismatch`, `consistent`。

- **基本内容**  
  编写 `src/layer4_contradiction/expert_rules.py`，根据四个模态的 `valence` 值（使用 `va_inter`）按照文档中的规则优先级进行判定，返回矛盾类型。

- **任务实施的思路和结构**  
  1. 输入：四个模态的 valence 值（文本、语音、宏表情、微表情）。  
  2. 规则实现（优先级从高到低）：  
     ```python
     if text_v > 0.6 and micro_v < -0.4: return "masking"
     if text_v > 0.3 and (speech_v < -0.4 or macro_v < -0.4): return "sarcasm"
     if abs(macro_v) < 0.2 and abs(micro_v) > 0.5: return "hidden_emotion"
     if same_sign(text_v, micro_v) and abs(text_v - micro_v) > 0.6: return "intensity_mismatch"
     return "consistent"
     ```  
  3. 辅助函数 `same_sign(a, b)` 判断符号相同（非负或非正）。  
  4. 规则阈值可从配置文件读取（`config/global.yaml` 或 `pipeline.yaml`），便于调优。  
  5. 同时返回 `involved_modalities`（产生矛盾的主要模态对），例如 `masking` 涉及 `['text','micro']`。  
  6. 该模块被L4主流程调用。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer4_contradiction/expert_rules.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L4.3 专家规则分类（含Python代码）。

- **需要参考的具体论文或者模型**  
  - 规则来自领域知识，文档中已明确定义。可参考 **Real-Time Micro-Expression Recognition with AU+GPT** 中的规则设计。

- **预估时间（AI/agent辅助）**

### 任务4.4：实现融合权重生成器（动态路由）

- **任务题目**  
  实现融合权重生成器，根据矛盾类型（及可选的矛盾强度、置信度）输出各模态的建议融合权重（4维，和为1）及路由置信度。

- **基本内容**  
  编写 `src/layer4_contradiction/weight_selector.py`，实现初版策略：查表（从 `weight_table.yaml` 读取矛盾类型到权重的映射）。同时计算 `routing_confidence`（基于矛盾强度的归一化值）。进阶预留神经网络接口。

- **任务实施的思路和结构**  
  1. 加载 `config/weight_table.yaml`，格式示例：  
     ```yaml
     masking: [0.1, 0.2, 0.2, 0.5]
     sarcasm: [0.4, 0.4, 0.1, 0.1]
     hidden_emotion: [0.1, 0.1, 0.1, 0.7]
     intensity_mismatch: [0.3, 0.2, 0.3, 0.2]
     consistent: [0.25, 0.25, 0.25, 0.25]
     ```  
  2. 权重顺序：`[text_weight, speech_weight, macro_weight, micro_weight]`。  
  3. `routing_confidence` 计算：`min(1.0, intensity / 1.2)` 或基于各模态置信度均值的组合（可配置）。  
  4. 提供 `get_weights(contradiction_type, intensity, confidences=None) -> (weights, confidence)` 函数。  
  5. 进阶：若配置 `weight_strategy: "small_nn"`，则调用一个小型神经网络（输入距离矩阵+各模态置信度+矛盾类型编码），初期不实现，仅预留接口。  
  6. 本模块输出 `suggested_fusion_weights` 和 `routing_confidence`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer4_contradiction/weight_selector.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L4.4 新增融合权重生成；`具体核心项目结构V2.0.docx` 中 `weight_selector.py` 和 `weight_table.yaml`。

- **需要参考的具体论文或者模型**  
  - **Real-Time Micro-Expression Recognition with AU+GPT**：路由思想。  
  - **Libra**（多模态大模型路由）提供动态权重参考。

### 任务4.5：实现矛盾结果数据结构及L4统一入口

- **任务题目**  
  实现 `ContradictionResult` 数据类（已在阶段0定义，但需完善），并编写 `run_l4(context)` 函数，串联距离计算、规则分类、权重生成，将结果存入 `DataContext.contradiction`。

- **基本内容**  
  编写 `src/layer4_contradiction/__init__.py` 中的 `run_l4` 函数，对每个片段调用L4各子模块，输出 `ContradictionResult`。同时支持片段级和整体级（整体矛盾汇总可选）。

- **任务实施的思路和结构**  
  1. 确认 `types.ContradictionResult` 字段完整：`contradiction_type`, `contradiction_intensity`, `involved_modalities`, `suggested_fusion_weights`, `routing_confidence`。  
  2. `run_l4(context)` 流程：  
     - 从 `context.segments` 获取片段列表。  
     - 对每个片段，从 `fragment.va_inter` 中取出四个模态的平均 `(v, a)`。  
     - 调用 `va_distance.py` 计算距离矩阵、最大距离及涉及模态对。  
     - 调用 `expert_rules.classify(...)` 得到矛盾类型。  
     - 调用 `weight_selector.get_weights(type, intensity, confidences)` 得到权重和置信度。  
     - 构造 `ContradictionResult` 对象，存入 `fragment.contradiction`。  
     - 同时可存储整体矛盾汇总（如矛盾事件列表）到 `context.contradiction_summary`（可选）。  
  3. 确保每个片段的结果可序列化。
