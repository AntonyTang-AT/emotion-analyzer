## 阶段5：L5 生成式报告（任务规划）

> 前置依赖：阶段4已完成（每个片段包含 `contradiction.suggested_fusion_weights`、矛盾类型等），阶段3已完成（片段列表及历史检索结果），L2的 `va_inter` 序列可用。
>
> 设计边界：L5 是**叙事层**。它只能解释 L4 已锁定的加权 VA、路由决策和证据，不得重新判断或覆盖上游 VA。默认使用本地 Qwen，云端 API 仅作可选后端。

### 任务5.1：实现Prompt模板管理

- **任务题目**  
  实现Prompt模板管理模块，集中管理片段级和整体级报告的Prompt模板，支持权重、VA轨迹、矛盾类型、历史相似片段等占位符的注入。

- **基本内容**  
  编写 `src/layer5_report/prompt_templates.py`，定义片段级和整体级报告的Prompt模板字符串，并提供函数 `format_segment_prompt` 和 `format_overall_prompt` 将变量填充到模板中。

- **任务实施的思路和结构**  
  1. 定义模板常量：  
     - `SEGMENT_PROMPT_TEMPLATE`：包含权重描述、**L4 已锁定的加权 VA**、路由决策、矛盾类型、历史相似片段等占位符。模板必须明确“只解释给定结果，不重新估计 VA 或主导情绪”。  
     - `OVERALL_PROMPT_TEMPLATE`：包含片段报告列表、矛盾事件汇总、平静指数/焦虑指数要求。参考文档：“请基于以下片段的情绪分析，给出整个对话的总结（150-300字）：\n[片段1报告]\n[片段2报告]\n...\n并计算平静指数（1-10）和焦虑指数（1-10）。”  
  2. 提供填充函数：  
     - `format_segment_prompt(weights, va_trajectory, contradiction_type, similar_history)` 返回字符串。  
     - `format_overall_prompt(segment_reports, contradiction_summary)` 返回字符串。  
  3. 模板中的权重格式化：将权重值转为自然语言描述（如“文本权重0.1，微表情权重0.5，请重点关注微表情”）。  
  4. 支持从配置文件读取自定义模板（便于后期调优），但模板校验必须拒绝“重新判断情绪”“修正上游 VA”等越权指令。  
  5. 注意：`similar_history` 需要格式化为易读文本（例如“历史片段1：悲伤情绪，相似度0.85”）。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer5_report/prompt_templates.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L5.1 和 L5.2 的Prompt模板；`论文总览.docx` 中 **NUS-Emo** 的指令模板设计。

- **需要参考的具体论文或者模型**  
  - **NUS-Emo**：情感-原因感知的指令模板，指导LLM输出结构化情感分析。  
  - **EmoLLM**：EmoPrompt（CoT推理引导），可借鉴其链式推理提示。

- **预估时间（AI/agent辅助）**  
  0.5 小时

### 任务5.2：实现片段级报告生成（调用LLM）

- **任务题目**  
  实现片段级报告生成模块，默认调用本地 `Qwen2.5-7B-Instruct`（INT4）根据 Prompt 生成自然语言情绪解读；DeepSeek/OpenAI API 作为可选后端。

- **基本内容**  
  编写 `src/layer5_report/segment_reporter.py`，对每个片段使用 `prompt_templates.format_segment_prompt` 构造Prompt，调用配置的LLM接口，返回30-80字的片段总结。

- **任务实施的思路和结构**  
  1. 新建统一 `llm_client.py`，封装 `call_llm(prompt, model_name, max_tokens=256, temperature=0.3)`，支持多种后端：  
     - OpenAI API（GPT-4o-mini）  
     - DeepSeek API  
     - 本地 Qwen（默认，通过 `transformers`、vLLM 或 `llama.cpp`）  
  2. `SegmentReporter` 类：  
     - `__init__(self, config)`：读取 `pipeline.yaml` 中 L5 配置（`llm_model`, `api_key`, `base_url` 等）。  
     - `generate_segment_report(fragment, context)`：  
       - 从 `fragment.contradiction` 获取权重、矛盾类型。  
       - 读取 L4 已锁定的加权 VA 轨迹；若尚未缓存，使用最终 `suggested_fusion_weights` 计算一次并锁定，不允许 LLM 修改。  
       - 从 `fragment.memory_retrieved` 取 Top-2 相似历史片段，格式化为文本。  
       - 调用模板填充得到prompt。  
       - 调用LLM获取响应，清洗后返回报告文本。  
  3. 缓存机制：避免重复生成相同片段的报告（可选）。  
  4. 错误处理：LLM 调用失败或超时时使用确定性的模板摘要；不得回退为 LLM 重算 VA。  
  5. 报告存入 `fragment.report` 字段，并在 `context.reports['segment']` 中按片段id存储。
  6. 增加后验一致性检查：报告情绪极性与 L4 加权 valence 明显相反时标记 `report_consistency=false`，并降级为模板摘要。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer5_report/segment_reporter.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L5.1；`具体核心项目结构V2.0.docx` 中 `segment_reporter.py`。

- **需要参考的具体论文或者模型**  
  - **NUS-Emo**：两阶段任务分解，先片段级后整体级。  
  - **Qwen2.5-7B-Instruct**：默认本地报告模型；DeepSeek-V3 / GPT-4o-mini 仅作可选远程后端。

- **预估时间（AI/agent辅助）**  
  1.5 小时（包括API集成和错误处理）

### 任务5.3：实现整体级报告生成

- **任务题目**  
  实现整体级报告生成，汇总所有片段报告、矛盾事件，计算平静指数和焦虑指数，生成150-300字的总体分析。

- **基本内容**  
  编写 `src/layer5_report/overall_reporter.py`，聚合片段报告、矛盾类型分布、VA时序统计，构造整体Prompt，调用LLM生成整体报告。同时依据规则计算平静/焦虑指数。

- **任务实施的思路和结构**  
  1. `OverallReporter` 类：  
     - `generate_overall_report(context)` 方法。  
  2. 输入数据：  
     - 所有片段的 `report` 文本列表。  
     - 矛盾事件汇总：各矛盾类型出现的次数、占比。  
     - 整体VA轨迹（加权平均后的时序）。  
  3. 计算平静指数和焦虑指数（Dual-layer VR）：  
     - 焦虑指数 = (恐惧片段占比×0.4 + 悲伤占比×0.3 + 愤怒占比×0.2 + 惊讶占比×0.1) × 10。  
     - 情绪标签可从片段报告中提取（使用简单关键词匹配，或从矛盾类型映射）。  
     - 平静指数 = 10 - 焦虑指数（近似）。  
  4. 构造Prompt：使用 `prompt_templates.OVERALL_PROMPT_TEMPLATE`，将片段报告列表、矛盾汇总、平静/焦虑指数填入。  
  5. 调用LLM获取整体报告文本，附加指数作为结构化字段。  
  6. 输出存入 `context.reports['overall']`（字典包含 `text`, `calm_index`, `anxiety_index`）。  
  7. 可选：不调用LLM，直接基于规则生成摘要（初期备选）。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer5_report/overall_reporter.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L5.2 和 Dual-layer VR 映射；`具体核心项目结构V2.0.docx` 中 `overall_reporter.py`。

- **需要参考的具体论文或者模型**  
  - **Dual-Layer Emotion Sensing in VR**：规则AU → 基本情绪 → 高阶情感维度（平静/焦虑）。  
  - 指数计算参考文档中的公式。

- **预估时间（AI/agent辅助）**  
  1 小时

### 任务5.4：实现可视化模块（VA曲线、热力图、权重时序图）

- **任务题目**  
  实现可视化模块，生成VA时序曲线（加权后）、模态一致性热力图、以及新增的模态信任权重时序堆叠图。

- **基本内容**  
  编写 `src/layer5_report/visualizer.py`，使用 matplotlib / plotly / seaborn 绘制三种图表，并保存为图片文件或base64编码供前端使用。

- **任务实施的思路和结构**  
  1. 函数 `plot_va_curve(segments, output_path)`：  
     - 横轴时间（片段开始到结束），纵轴 valence（红）和 arousal（蓝）。  
     - 对每个时间点（或片段内插值），使用各片段的 `suggested_fusion_weights` 加权平均 `va_inter` 值。  
     - 绘制曲线，添加图例、网格。  
  2. 函数 `plot_consistency_heatmap(distance_matrix, output_path)`：  
     - 输入4×4距离矩阵（归一化到[0,1]）。  
     - 使用 `seaborn.heatmap`，标注模态名称。  
     - 对角线为1.0（自相似）。  
  3. 函数 `plot_weight_timeline(segments, output_path)`：  
     - **新增**：堆叠面积图，横轴为片段索引或时间中点，纵轴为权重（0~1）。  
     - 每个模态一条堆叠区域（文本、语音、宏表情、微表情）。  
     - 使用 `matplotlib.pyplot.stackplot` 或 `plotly` 绘制。  
     - 便于观察矛盾场景下系统对不同模态信任的变化。  
  4. 所有图表支持保存到文件（PNG/PDF）或返回base64供Web前端使用。  
  5. 集成到L5统一入口中，可配置开关。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer5_report/visualizer.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L5.3 可视化输出，及“新增模态信任权重时序图”；`接口与web端相关项目结构V2.0.docx` 中 `charts.js` 的 `drawWeightChart` 但这里是后端生成静态图。

### 任务5.6：实现L5层统一入口（run_l5）

- **任务题目**  
  实现L5层统一调用入口，顺序执行片段报告生成、整体报告生成、可视化图表生成，并将结果存入 `DataContext.reports`。

- **基本内容**  
  编写 `src/layer5_report/__init__.py` 中的 `run_l5(context)` 函数，调用上述各模块，更新context。

- **任务实施的思路和结构**  
  1. `run_l5(context)` 流程：  
     - 从 `context.segments` 获取片段列表，对每个片段调用 `SegmentReporter.generate_segment_report`，将结果存入 `fragment.report`。  
     - 调用 `OverallReporter.generate_overall_report`，存入 `context.reports['overall']`。  
     - 调用 `visualizer` 的三个绘图函数，将图片路径或base64存入 `context.reports['figures']`。  
     - 调用 `generate_statistical_summary` 存入 `context.reports['summary']`。  
  2. 支持配置开关：`pipeline.yaml` 中 L5 的 `enabled` 控制是否执行，以及 `save_figures` 是否保存到磁盘。  
  3. 异常处理：某一子模块失败不影响其他（如LLM调用失败则报告置为默认文本）。
