### 任务6.1：实现特征聚合器（从 L1-L5 提取人格预测特征）

- **任务题目**  
  实现特征聚合器，从 `DataContext` 和 `segments` 中提取约30-60维的特征向量，包括时序统计、矛盾统计、语音行为、响应延迟、个性化偏移、纯视觉旁路特征等。

- **基本内容**  
  编写 `src/layer6_personality/feature_aggregator.py`，聚合以下特征组：  
  1. 加权后的VA时序统计（均值、标准差、偏度、自相关系数）  
  2. 矛盾统计（各矛盾类型次数/占比、强度均值、平均路由置信度、各矛盾类型下的平均权重分布）  
  3. 语音行为（平均语速、音调标准差、微笑AU12强度、微表情恐惧次数）  
  4. 响应延迟（平均回答延迟时间，需从对话结构解析）  
  5. 个性化偏移（最终采用的 ΔVA 向量）  
  6. 纯视觉旁路特征（微表情原始特征平均强度、事件频率、DGM位移场能量等）

- **任务实施的思路和结构**  
  1. 输入：`DataContext` 对象（包含 `segments`、`features`、`raw_visual_features`、`contradiction` 各片段等）。  
  2. 特征抽取函数分步实现：  
     - `_extract_weighted_va_stats(context)`：  
       遍历片段，对每个片段内的 `va_inter` 时间序列用 `suggested_fusion_weights` 加权平均，得到每时刻的综合VA，再计算整体均值、标准差、偏度（scipy.stats.skew）、自相关系数（pandas autocorr 滞后1-5）。  
     - `_extract_contradiction_stats(context)`：  
       统计各片段 `contradiction_type` 计数、占比；计算所有矛盾强度的均值；各矛盾类型下权重的分布（平均权重向量）。  
     - `_extract_speech_behavior(context)`：  
       从 L1 语音特征中提取语速（总字数/总时长，需文本特征辅助）、音调标准差（从F0序列计算）、微笑强度（从宏表情或AU12中提取）、微表情恐惧次数（从微表情事件中统计）。  
     - `_extract_response_delay(context)`：  
       若对话有轮次结构（用户语句之间），计算平均回答延迟（从上一句结束到本句开始的时间差）。初期可设为0或跳过。  
     - `_extract_personalization_offset(context)`：  
       从用户数据库或 `context.metadata` 中获取用户最终的 ΔVA 向量（8维）。  
     - `_extract_raw_visual_features(context)`：  
       从 `context.raw_visual_features['micro']` 中计算平均强度、峰值、事件频率、DGM位移场能量（若可用）。  
  3. 将所有特征拼接成1维数组，记录特征名称列表，并标准化（保存均值和标准差）。  
  4. 提供 `aggregate(context) -> np.ndarray` 函数，以及 `get_feature_names() -> List[str]`。  
  5. 特征数量控制在30-60维，初期可提取约40维。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer6_personality/feature_aggregator.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L6.1 特征抽取（含加权统计、矛盾统计、纯视觉特征）；`接口与web端相关项目结构V2.0.docx` 中 `feature_aggregator.py`。

- **需要参考的具体论文或者模型**  
  - **Detection of arousal and valence from facial expressions and physiological responses**：MRMR特征选择思想，可用于后续降维。  
  - **Data-constrained personalization**：个性化偏移的使用。  
  - 无直接模型。

- **预估时间（AI/agent辅助）**  
  2 小时

### 任务6.2：实现 LightGBM 多输出回归模型

- **任务题目**  
  实现基于 LightGBM 的多输出回归模型，用于预测大五人格分数（O, C, E, A, N），范围1-10。

- **基本内容**  
  编写 `src/layer6_personality/lightgbm_regressor.py`，封装训练、预测、评估功能。支持多输出回归（每个维度独立或联合模型），损失函数MAE，并可选择多输出相关性正则项。

- **任务实施的思路和结构**  
  1. 类 `LightGBMPersonalityRegressor`：  
     - `__init__(self, config)`：读取模型超参数（n_estimators, learning_rate, max_depth 等）。  
     - `train(self, X, y)`：X 为特征矩阵（n_samples × n_features），y 为 (n_samples × 5)。使用 `lightgbm.LGBMRegressor` 的多输出模式（设置 `objective='regression'`，`metric='mae'`，`multi_output=True`）。  
     - 可选：为每个维度单独训练模型（简单但忽略相关性）。  
     - 实现交叉验证评估。  
     - `predict(self, X)`：返回 (n_samples × 5) 预测值。  
     - `save_model(path)` / `load_model(path)`。  
  2. 训练脚本 `scripts/train_personality.py` 调用该模块。  
  3. 特征标准化：使用 `sklearn.preprocessing.StandardScaler` 拟合训练集，保存 scaler。  
  4. 损失函数可用 MAE，目标 ≤0.8。  
  5. 可选：多输出相关性正则项 `λ * Σ|ρ_pred - ρ_true|`，可在训练后计算，不强制。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer6_personality/lightgbm_regressor.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L6.2 回归模型（LightGBM + 多输出相关性）。

- **需要参考的具体论文或者模型**  
  - **LightGBM**：`LightGBM: A Highly Efficient Gradient Boosting Decision Tree` (Ke et al., 2017)。  
  - 多输出回归标准方法。

- **预估时间（AI/agent辅助）**  
  1.5 小时

### 任务6.3：实现高斯过程回归（GP）可选概率增强

- **任务题目**  
  实现高斯过程回归作为可选的概率模型，输出预测均值 + 方差（置信区间）。

- **基本内容**  
  编写 `src/layer6_personality/gp_regressor.py`，使用 `sklearn.gaussian_process.GaussianProcessRegressor` 或 `GPy`，为每个大五维度独立建模（因为多输出GP较复杂），输出均值和标准差。

- **任务实施的思路和结构**  
  1. 类 `GPPersonalityRegressor`：  
     - `__init__(self, config)`：选择核函数（RBF + WhiteKernel）。  
     - `train(self, X, y)`：为每个维度独立训练一个GP模型。  
     - `predict(self, X)`：返回 (mean, std) 两个数组，shape (n_samples, 5)。  
  2. 由于GP计算复杂度高，仅用于小样本或作为对比。  
  3. 在配置中通过 `model_type: "gp"` 启用。  
  4. 训练数据较少时（< 100）可使用GP。  
  5. 提供保存/加载模型功能（使用 `pickle` 或 `joblib`）。

### 任务6.4：实现大模型零样本备用（LLM fallback）

- **任务题目**  
  实现大模型零样本备用方案，当特征不足或模型未训练时，调用LLM（GPT-4o）进行零样本人格推断。

- **基本内容**  
  编写 `src/layer6_personality/llm_fallback.py`，构造包含行为特征摘要的Prompt，调用LLM返回大五人格分数（1-10）及置信度区间。

- **任务实施的思路和结构**  
  1. 函数 `llm_personality_inference(context, feature_summary)`：  
     - 从 `context` 中提取简洁摘要：矛盾类型分布、整体情绪基调、语音速度、微表情强度等。  
     - 构建Prompt：“你是一个心理学专家，根据以下用户行为特征，推断其大五人格（开放性、尽责性、外向性、宜人性、神经质），每个维度1-10分。输出格式JSON: {‘O’: , ‘C’: , ‘E’: , ‘A’: , ‘N’: }。行为特征：...”  
     - 调用配置的LLM（如GPT-4o）。  
     - 解析JSON返回结果，若失败则返回默认值（如5分）。  
  2. 提供置信度区间（基于LLM的logits或固定值，如±0.8）。  
  3. 当 `len(features) < min_fragments`（如少于30个片段）或 LightGBM 模型不存在时启用。  
  4. 在 `run_l6` 中作为备用。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer6_personality/llm_fallback.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L6.2 冷启备选；`论文总览.docx` 中 **EmoLLM** 的 EmoPrompt 思路。

- **需要参考的具体论文或者模型**  
  - **GPT-4o** 作为基座模型。  
  - 零样本人格推断相关研究（无特定论文）。

- **预估时间（AI/agent辅助）**  
  1 小时

### 任务6.5：实现训练数据准备脚本

- **任务题目**  
  编写训练数据准备脚本，支持自建小样本标注、Friends弱标注、GPT-4o伪标注等多种方式，生成特征-人格标签对。

- **基本内容**  
  编写 `scripts/prepare_personality_data.py`，从原始对话视频/音频中提取特征（调用L1-L4流水线），并从外部标签（BFI问卷、文献标签、GPT标注）获取人格分数，生成训练集 `X, y` 并保存为 `.npy` 或 `.csv`。

- **任务实施的思路和结构**  
  1. 三种数据来源：  
     - 自建小样本：30-50人，完成标准化对话（录制视频）并填写BFI-10量表，运行流水线提取特征，得到特征-标签对。  
     - Friends弱标注：使用Friends角色的已知人格研究（如文献中给出的角色OCEAN分数），从多轮对话中抽取特征，生成伪标签。  
     - GPT-4o伪标注：对无标签对话，调用LLM进行零样本人格标注，再在少量人工标注子集上校准。  
  2. 脚本流程：  
     - 读取配置文件中的数据路径。  
     - 对每个样本（视频路径 + 标签文件），调用 `run_pipeline` 获取特征聚合结果（可复用L6的特征聚合函数）。  
     - 合并所有样本的特征和标签，进行标准化，保存为 `train_features.npy`, `train_labels.npy`, `test_*.npy`。  
  3. 支持交叉验证划分。  
  4. 记录特征名称列表。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`scripts/prepare_personality_data.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L6.4 训练数据；`接口与web端相关项目结构V2.0.docx` 中 `scripts/` 结构。

- **需要参考的具体论文或者模型**  
  - **Friends角色人格研究**：文献中可能提供角色OCEAN分数（如大五人格与角色分析）。  
  - 无直接模型。

- **预估时间（AI/agent辅助）**

### 任务6.6：实现L6层统一入口（run_l6）

- **任务题目**  
  实现L6层统一调用入口，调用特征聚合器，根据配置选择回归模型（LightGBM / GP / LLM fallback），输出人格分数及置信区间、行为证据文本，存入 `DataContext.personality`。

- **基本内容**  
  编写 `src/layer6_personality/__init__.py` 中的 `run_l6(context)` 函数。

- **任务实施的思路和结构**  
  1. 检查 `context.segments` 数量，若少于配置的 `min_fragments`（如30），则直接跳转到 LLM fallback。  
  2. 调用 `feature_aggregator.aggregate(context)` 得到特征向量。  
  3. 根据 `pipeline.yaml` 中的 `model_type`：  
     - `"lightgbm"`：加载预训练 LightGBM 模型和 scaler，预测并输出5个分数。  
     - `"gp"`：加载 GP 模型，输出均值 + 标准差作为置信区间。  
     - 若模型不存在或加载失败，调用 `llm_fallback`.  
  4. 构造 `PersonalityResult` 结构：包含 `scores`（字典）、`confidence_interval`（列表）、`behavioral_evidence`（简短描述文本，可从特征汇总生成）。  
  5. 存入 `context.personality`。  
  6. 如果 `L6.enabled` 为 false，则跳过。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer6_personality/__init__.py` 中的 `run_l6`  
  - 参考：`技术路线总指南V2.0.docx` 中 L6.3 输出格式；`具体核心项目结构V2.0.docx` 中 `lightgbm_regressor.py` 和 `__init__.py`。

- **需要参考的具体论文或者模型**  
  无。

- **预估时间（AI/agent辅助）**  
  1 小时

### 任务6.7：实现评估脚本与指标计算

- **任务题目**  
  编写评估脚本，计算人格回归模型的 MAE、Pearson 相关系数、ICC（重测信度）。

- **基本内容**  
  编写 `scripts/evaluate_personality.py`，在测试集上运行训练好的模型，计算各项指标，输出报告。

- **任务实施的思路和结构**
