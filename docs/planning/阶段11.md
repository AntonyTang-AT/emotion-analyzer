### 任务11.1：编写 Dockerfile（API 和 Web）

- **任务题目**  
  编写后端 API 和前端 Web 服务的 Dockerfile，支持一键构建镜像。

- **基本内容**  
  创建 `Dockerfile.api` 和 `Dockerfile.web`。  
  - API 镜像：基于 Python 3.10-slim，安装依赖，复制 core/ 和 server/ 代码，暴露 8000 端口，启动命令 `uvicorn server.main:app --host 0.0.0.0 --port 8000`。  
  - Web 镜像：基于 nginx:alpine，复制 web/ 目录下的静态文件到 `/usr/share/nginx/html`，暴露 80 端口。

- **任务实施的思路和结构**  
  1. API Dockerfile：  
     ```dockerfile
     FROM python:3.10-slim
     WORKDIR /app
     COPY requirements.txt .
     RUN pip install --no-cache-dir -r requirements.txt
     COPY core/ ./core/
     COPY server/ ./server/
     COPY config/ ./config/
     COPY models/ ./models/   # 预训练模型权重
     EXPOSE 8000
     CMD ["uvicorn", "server.main:app", "--host", "0.0.0.0", "--port", "8000"]
     ```  
  2. Web Dockerfile：  
     ```dockerfile
     FROM nginx:alpine
     COPY web/ /usr/share/nginx/html
     EXPOSE 80
     ```  
  3. 优化镜像大小：多阶段构建（可选），使用 .dockerignore 排除不需要的文件。  
  4. 确保 API 镜像可以读取外部配置文件和环境变量（如数据库连接、JWT secret）。  
  5. 提供构建脚本 `scripts/build_images.sh`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`Dockerfile.api`, `Dockerfile.web`, `.dockerignore`, `scripts/build_images.sh`  
  - 参考：无特定文档。

### 任务11.2：编写 docker-compose.yml（完整服务编排）

- **任务题目**  
  编写 docker-compose 文件，编排 API、Web、PostgreSQL、Chroma（向量数据库）以及可选的 Redis（缓存）等服务。

- **基本内容**  
  创建 `docker-compose.yml`，定义以下服务：  
  - `api`：构建后端镜像，依赖 db、chroma，暴露 8000。  
  - `web`：构建前端镜像，暴露 80。  
  - `db`：使用 postgres:15，持久化数据卷，初始化数据库。  
  - `chroma`：使用 chromadb/chroma 官方镜像，持久化向量数据。  
  - `nginx`（可选）：反向代理 API 和 Web。  
  配置网络、环境变量（从 `.env` 文件加载）。

- **任务实施的思路和结构**  
  1. 服务依赖：api 等待 db 和 chroma 就绪（使用 `depends_on` 和 `healthcheck`）。  
  2. 环境变量：数据库 URL、JWT_SECRET、CHROMA_HOST 等。  
  3. 卷挂载：  
     - db: `postgres_data:/var/lib/postgresql/data`  
     - chroma: `chroma_data:/chroma/chroma`  
     - api: 可挂载配置文件或模型目录（便于更新）  
  4. 提供 `docker-compose.override.yml` 用于开发环境（热重载）。  
  5. 启动命令：`docker-compose up -d`。

- **主要针对修改哪些具体的文件，需要参考哪些文件**

### 任务11.3：配置 Nginx 反向代理与 HTTPS（可选）

- **任务题目**  
  配置 Nginx 作为前端入口和反向代理，将 `/api` 请求转发到后端 API，并配置 HTTPS（使用 Let‘s Encrypt）。

- **基本内容**  
  编写 `nginx/nginx.conf`，实现：  
  - 静态文件服务（前端）  
  - 反向代理 `/api` 到 API 容器  
  - WebSocket 代理（`/ws`）  
  - HTTPS 配置（证书路径）

- **任务实施的思路和结构**  
  1. 在 `docker-compose.yml` 中添加 `nginx` 服务，暴露 80 和 443，依赖于 api 和 web。  
  2. `nginx.conf` 配置：  
     - `location /` → 代理到 web 容器（或直接 serve 前端文件）  
     - `location /api/` → `proxy_pass http://api:8000/api/`  
     - `location /ws` → `proxy_pass http://api:8000/ws`，并设置 `Upgrade` 头支持 WebSocket。  
  3. 使用 Certbot 自动申请证书，或手动挂载证书文件。  
  4. 提供脚本 `scripts/setup_https.sh` 自动化证书获取。
