### 任务0.4：定义核心数据上下文 `DataContext`

- **基本内容**  
  在 `src/core/context.py` 中实现 `DataContext` 类，用于贯穿整个流水线的数据传递，包括原始路径、特征、两套VA预测、片段列表、矛盾结果、报告、人格结果，以及纯视觉旁路特征、记忆检索结果等（根据技术文档定义）。

- **任务实施的思路和结构**  
  1. 定义类属性：`metadata`（会话ID、用户ID、时间戳）、`raw_data`（视频/音频路径）、`features`（L1输出）、`raw_visual_features`（纯视觉旁路字典）、`va_self_predictions`、`va_inter_predictions`、`segments`、`memory_retrieved`、`contradiction`、`reports`、`personality`。  
  2. 提供 `set_stage(stage_name, data)` 和 `get_stage(stage_name)` 方法，便于按阶段存取数据。

  4. 可集成 `pydantic` 进行类型校验（可选）。  
  5. 参考技术文档中“层间通信：统一数据上下文”章节的字段定义。

### 任务0.5：定义接口抽象类（`interfaces.py`）与自定义类型（`types.py`）

- **基本内容**  
  在 `src/core/interfaces.py` 中定义各层（L1~L6）的抽象基类（如 `FeatureExtractor`、`VAPredictor`、`SegmentController`、`ContradictionDetector`、`ReportGenerator`、`PersonalityRegressor`），统一 `run` 或 `process` 接口。  
  在 `src/core/types.py` 中定义自定义数据类型，如 `VAConfidence`、`Fragment`、`ContradictionResult`（含权重字段）、`PersonalityResult` 等。

- **任务实施的思路和结构**  
  1. 继承 `ABC` 和 `abstractmethod`。  
  2. `FeatureExtractor` 定义 `extract(data_context) -> FeatureDict`。  
  3. `VAPredictor` 定义 `predict_self(features) -> VAConfidence` 和 `predict_inter(features) -> VAConfidence`（两分支）。  
  4. `ContradictionResult` 使用 `@dataclass`，字段包括：`contradiction_type`、`intensity`、`involved_modalities`、`suggested_fusion_weights`（List[float] 长度4）、`routing_confidence`。  
  5. `Fragment` 包含 `id`、`start_time`、`end_time`、`va_self`（dict）、`va_inter`（dict）、`contradiction`（可选）。  
  6. 所有类型尽量使用 `pydantic` 或 `dataclass`，便于序列化。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/core/interfaces.py`、`src/core/types.py`  
  - 参考：`具体核心项目结构V2.0.docx` 中的 `interfaces.py` 和 `types.py` 说明；`技术路线总指南V2.0.docx` 中 L4 输出的 `ContradictionResult` 定义

- **需要参考的具体论文或者模型**  
  - **EmoLILMs**：多任务输出结构（同时输出 valence/arousal/confidence）  
  - **FRL-DGT**：微表情特征输出的数据结构（位移场、AU强度）  
  - **NUS-Emo**：两级报告的数据组织方式

- **预估时间（AI/agent辅助）**  
  1 小时

### 任务0.6：搭建基础工具函数模块

- **基本内容**  
  编写 `src/utils/audio_utils.py`、`src/utils/video_utils.py`，提供音频/视频的加载、帧提取、时间戳对齐、重采样等基础功能，供 L1 特征提取器使用。

- **任务实施的思路和结构**  
  1. `audio_utils.py`：使用 `librosa` 或 `torchaudio` 加载音频，支持重采样到16kHz，提取时长、采样率，提供分帧函数。  
  2. `video_utils.py`：使用 `cv2` 或 `decord` 读取视频，按时间戳提取帧，支持将视频帧与音频时间戳对齐。  
  3. 提供通用函数如 `get_audio_duration(path)`、`extract_frames_at_times(video_path, times)`。  
  4. 注意处理不同视频编码格式，统一输出为 numpy 数组。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/utils/audio_utils.py`、`src/utils/video_utils.py`  
  - 参考：无直接文档，属于通用工具，但需确保输出格式与 L1 提取器要求一致（如帧率、时间戳精度）
