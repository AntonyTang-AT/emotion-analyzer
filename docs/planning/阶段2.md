### 任务2.1：实现VA预测器基类与两分支MLP模型

- **任务题目**  
  实现 `BasePredictor` 基类及两分支MLP模型（`TwoBranchMLP`），用于每个模态的VA预测（自建模分支 `predict_self` 和交互分支 `predict_inter`）。

- **基本内容**  
  编写 `src/layer2_predict/base_predictor.py` 定义抽象基类，声明 `predict_self(features) -> VAConfidence` 和 `predict_inter(features) -> VAConfidence`。  
  编写 `src/layer2_predict/two_branch_mlp.py` 实现 `TwoBranchMLP` 类，包含共享底层（Linear+ReLU）和两个独立顶层（各输出 v, a, conf），输出范围 `[-1,1]`。

- **任务实施的思路和结构**  
  1. `base_predictor.py`：继承 `ABC`，`@abstractmethod` 定义两个预测方法；`VAConfidence` 使用阶段0定义的 `types`（如 `NamedTuple` 或 `pydantic` 模型）。  
  2. `two_branch_mlp.py`：  
     - 构造函数参数：`input_dim`, `shared_dim=256`, `hidden_dim=128`, `dropout=0.2`。  
     - 网络结构：`shared_fc = Linear(input_dim, shared_dim) + ReLU`；  
       `self_branch = Sequential(Linear(shared_dim, hidden_dim), ReLU, Dropout(p), Linear(hidden_dim, 3))`；  
       `inter_branch` 结构相同但独立参数。  
     - 输出使用 `tanh` 将 v,a 限制在 `[-1,1]`，`conf` 使用 `sigmoid` 输出 `[0,1]`。  
     - `forward` 方法接受 `features` 并返回两个分支的输出（可分别调用 `predict_self`/`predict_inter`）。  
  3. 模型权重初始化（xavier 或 kaiming）。  
  4. 支持从配置文件加载预训练权重（`models.yaml` 中指定路径）。  
  5. 单元测试：用随机输入测试输出形状和范围。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer2_predict/base_predictor.py`、`src/layer2_predict/two_branch_mlp.py`  
  - 参考：`技术路线总指南V2.0.docx` 中 L2.1 章节；`具体核心项目结构V2.0.docx` 中 `layer2_predict/` 和 `two_branch_mlp.py` 说明。

- **需要参考的具体论文或者模型**  
  - **EmoLILMs**：多任务输出结构（同时预测 valence, arousal, confidence）  
  - 无直接对应论文，属于基础 MLP 设计。

- **预估时间（AI/agent辅助）**  
  1 小时

### 任务2.3：实现L2层统一调用入口（单模态预测）

- **任务题目**  
  实现L2层处理函数，对每个模态的特征分别调用预测器，得到两套VA预测，存入 `DataContext`。

- **基本内容**  
  编写 `src/layer2_predict/predictor.py`（或直接在 `__init__.py` 中），实现 `run_l2(context)` 函数，遍历所有已注册的模态，从 `context.features` 中取出特征，调用 `predict_self` 和 `predict_inter`，将结果存入 `context.va_self_predictions` 和 `context.va_inter_predictions`。

- **任务实施的思路和结构**  
  1. 函数签名：`def run_l2(context: DataContext) -> DataContext`。  
  2. 对于每个模态：  
     - 获取特征列表（可能每个片段或每秒一个），目前 L1 输出可能是时间序列。需要对齐到同一时间轴（每秒一个）。  
     - 可简化：初期对每个时间步独立预测，特征为每秒向量。  
     - 调用 `predictor.predict_self(feature)` 和 `predict_inter(feature)`，得到 `VAConfidence` 对象。  
     - 存入 `context.va_self_predictions[modal_name] = list_of_VAConfidence`（按时间顺序）。  
  3. 若某模态特征缺失或预测器未注册，记录警告并跳过。  
  4. 确保所有模态的时间轴对齐（使用统一的时间戳数组）。  
  5. 配置 `pipeline.yaml` 中 L2 的 `enabled` 控制是否执行。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer2_predict/__init__.py` 中的 `run_l2` 函数  
  - 参考：`技术路线总指南V2.0.docx` 中 L2 数据流；`具体核心项目结构V2.0.docx` 中 `layer2_predict` 结构。

- **需要参考的具体论文或者模型**  
  无。

### 任务2.4：实现多任务联合优化训练脚本（两分支训练）

- **任务题目**  
  编写训练脚本 `scripts/train_models.py`，支持两分支MLP的多任务联合优化，可分别设置自建模分支和交互分支的损失权重，并使用可学习损失权重（EmoLILMs思想）。

- **基本内容**  
  实现训练循环，使用包含单模态VA标注的数据集（或带有多模态对齐标注的数据）训练每个模态的 `TwoBranchMLP`。支持自建模分支使用单模态标注，交互分支使用多模态对齐标注。损失函数为 MSE + 二元交叉熵（置信度），并支持可学习损失权重。

- **任务实施的思路和结构**  
  1. 数据加载：需要构造数据集，每个样本包含（特征向量，valence_self, arousal_self, conf_self, valence_inter, arousal_inter, conf_inter）。初期可模拟或使用公开数据集（如 IEMOCAP 提供 VA 标注）。  
  2. 模型：为每个模态独立实例化 `TwoBranchMLP`。  
  3. 损失函数：  
     - 自建模分支损失：`L_self = MSE(v_self_pred, v_self_true) + MSE(a_self_pred, a_self_true) + BCE(conf_self_pred, conf_self_true)`。  
     - 交互分支损失：`L_inter` 同理。  
     - 总损失：`L_total = λ_self * L_self + λ_inter * L_inter`。`λ` 可为固定值（如0.3/0.7）或可学习参数（使用 `torch.nn.Parameter`，初始1.0）。  
  4. 优化器：Adam。  
  5. 训练循环中，根据配置的 epoch、batch size 进行训练，保存最佳模型权重到 `models/` 目录。  
  6. 支持从检查点恢复训练。  
  7. 在配置文件中定义各模态的训练参数（学习率、损失权重、数据路径等）。  
  8. 添加验证集评估（MAE、相关系数）。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`scripts/train_models.py`，以及可能的 `src/layer2_predict/losses.py`（可学习损失权重）  
  - 参考：`技术路线总指南V2.0.docx` 中 L2.2 多任务联合优化；`论文总览.docx` 中 **EmoLILMs** 的多任务损失设计。

- **需要参考的具体论文或者模型**  
  - **EmoLILMs**：多任务指令数据集和可学习损失权重。  
  - 损失函数设计可参考 `Multi-Task Learning Using Uncertainty to Weigh Losses for Scene Geometry and Semantics` (Kendall et al., 2018)。

- **预估时间（AI/agent辅助）**  
  2 小时（不含数据准备）

### 任务2.5：实现可选增强：DTFE（后期优化，初期可跳过）

- **任务题目**  
  实现动态Token特征提取器（DTFE）作为语音/微表情模态的可选增强模块。

- **基本内容**  
  根据论文《Structured multi-modal representation learning for affective state (DTFE)》，实现动态Token特征提取器，将长序列语音或微表情特征转化为 token 序列，并通过跨序列交互（CSI）和时序频谱建模（ISI）提升表示质量。初期可跳过，作为后期优化任务。

- **任务实施的思路和结构**  
  1. 在 `src/layer2_predict/dtfe_enhancer.py` 中实现 DTFE。  
  2. 主要组件：  
     - 动态Token投影：将原始时域信号（如语音帧序列）通过可学习线性投影得到 token 序列。  
     - 跨序列交互（CSI）：使用多头注意力建模不同通道（或不同模态 token）间的关系。  
     - 时序频谱建模（ISI）：对 token 序列做 FFT，应用可学习频域门控，再逆变换。  
  3. 可集成到 L2 预测器中，作为特征预处理模块。  
  4. 配置开关 `dtfe_enabled` 在 `features.yaml` 中。  
  5. 由于复杂度高，初期不实现，仅预留接口。

- **主要针对修改哪些具体的文件，需要参考哪些文件**  
  - 新建：`src/layer2_predict/dtfe_enhancer.py`（后期）  
  - 参考：`技术路线总指南V2.0.docx` 中 L2.3；`论文总览.docx` 中 **DTFE** 论文。

- **需要参考的具体论文或者模型**  
  - **DTFE**：`Structured multi-modal representation learning for affective state`（详细方法见论文）。

- **预估时间（AI/agent辅助）**  
  暂不预估（后期优化）
